### 2. **修改端口的方法**

#### **方法一：修改Dockerfile（推荐）**
编辑 <mcfile name="Dockerfile" path="/suanfa-1/jzp/wind_whisper_rag_system/Dockerfile"></mcfile> 第69行：

```bash
# 将8003改为8000
echo 'command=uvicorn main:app --host 0.0.0.0 --port 8000 --reload' >> /etc/supervisor/conf.d/supervisord.conf
```

同时修改第77行的EXPOSE：
```bash
# 将8003改为8000
EXPOSE 8000
```

#### **方法二：修改main.py的默认端口**
编辑 <mcfile name="main.py" path="/suanfa-1/jzp/wind_whisper_rag_system/main.py"></mcfile> 第120行：

```python
# 将默认端口从8000改为你想要的端口
port = int(os.getenv("PORT", "8000"))  # 这里改成你想要的默认端口
```

### 4. **修改步骤（推荐方案）**

如果你想将端口改为8000：

1. **修改Dockerfile**：
```bash
# 第69行改为：
echo 'command=uvicorn main:app --host 0.0.0.0 --port 8000 --reload' >> /etc/supervisor/conf.d/supervisord.conf

# 第77行改为：
EXPOSE 8000
```

2. **修改docker-compose.yml的端口映射**：
```yaml
ports:
  - "8000:8000"  # 将8003改为8000
```

3. **重新构建镜像**：
```bash
docker-compose down
docker-compose build
docker-compose up -d
```

### 5. **环境变量方式（最灵活）**

你也可以通过环境变量来控制端口，在 <mcfile name=".env" path="/suanfa-1/jzp/wind_whisper_rag_system/.env"></mcfile> 中添加：
```env
PORT=8000
HOST=0.0.0.0
```

然后修改Dockerfile中的supervisor配置使用环境变量。

**总结：** 当前应用运行在8003端口是因为Dockerfile中硬编码了这个端口。你可以通过修改Dockerfile第69行来改变启动端口。
